# ビルドライブラリテスト

このプロジェクトでは、ビルドされたSymbol Sign TXライブラリをHTMLから使用するためのテストページが用意されています。

## テストページ一覧

### 1. 高機能テストページ (`build-test.html`)
- ライブラリの読み込み状況確認
- 基本的なトランザクション作成テスト
- 複数パターンテスト（6種類のテストケース）
- パフォーマンステスト（100回実行）
- 美しいUIデザイン

### 2. シンプルテストページ (`simple-build-test.html`)
- 基本的な機能テストに特化
- 軽量でシンプルなUI
- エラーハンドリング付き

### 3. 既存のテストページ
- `index.html` - メインのテストページ
- `simple.html` - シンプルなテストページ

## テスト実行方法

1. プロジェクトをビルド:
```bash
yarn build
```

2. テストサーバーを起動:
```bash
yarn test:server
```

3. ブラウザでアクセス:
- http://localhost:3000/test/build-test.html （高機能テスト）
- http://localhost:3000/test/simple-build-test.html （シンプルテスト）
- http://localhost:3000/test/index.html （メインテスト）
- http://localhost:3000/test/simple.html （シンプルテスト）

## ビルドとテスト一括実行

```bash
yarn test:build
```

このコマンドで、ビルドとテストサーバー起動を一度に実行できます。

## テスト内容

### 基本機能テスト
- ライブラリの動的読み込み（ES Module）
- トランザクション作成機能
- ネットワーク設定（mainnet/testnet）
- メッセージ設定

### 高度なテスト（`build-test.html`）
- 複数パターンテスト
  - 英語メッセージ
  - 日本語メッセージ
  - 特殊文字
  - 空文字
  - 長文（100文字）
- パフォーマンステスト
  - 100回連続実行
  - 実行時間計測
  - 統計情報（平均、最速、最遅、中央値、95パーセンタイル）
  - スループット計算

### 機能
- ✅ ライブラリ読み込み状況表示
- ✅ リアルタイムエラーハンドリング
- ✅ 実行時間計測
- ✅ トランザクションHexのクリップボードコピー
- ✅ レスポンシブデザイン
- ✅ 美しいUI/UX

## 技術仕様

- **ライブラリ読み込み**: ES Module の動的import
- **ビルド出力**: `dist/symbol-sign-tx.js`
- **フォーマット**: ESM (ES Module)
- **ブラウザ対応**: モダンブラウザ（ES2022+）
- **依存関係**: symbol-sdk（外部依存として除外済み）
